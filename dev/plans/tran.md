# instructions for "Add Client" feature
- This is a three step process:
    1. Create a new client in the `service_plus_client.client` table.
        - Take field details from server's service_plus_client.sql file.
    2. Create a new database with the `security` schema.
        - database name should be service_plus_<client_code>
    3. Create a new admin user in the `security` schema.
        - Only valid email is required. Username and password should be generated by the server and sent to the email account.
        - Admin should be able to login and change username and password.
- All the above steps can be done in disassociated manner. 
- The client overview table should be updated with the new client details.
- The client overview table actions should have "create database" and "create admin user" and "modify admin user" buttons.
- The "create database" button should create a new database with the `security` schema.
- The "create admin user" button should create a new admin user in the `security` schema.
- The "create database" button should be disabled if the database is already created.
- The "modify admin user" button should allow to enable / disable / delete the admin user details.
- There can be multiple admin users for a client.
- genericUpdate mutation can be used to create a new client.
- New mutation should be created to create a new database.
- genericUpdate mutation can be used to create a new admin user.
- genericUpdate mutation can be used to modify an admin user.
