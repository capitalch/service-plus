# Opening jobs: Change behavior and functionality
- All opening jobs to be entered as is_opening_job = true. The view grid should be filtered accordingly.
- New opening job will be having a new job no, auto generated by server as at present.
- The job no provided by user should go to alternate_job_no, instead of job_no
- Opening job can be identified as is_opening_job = true
- In single job: view: the opening jobs should be filtered out and won't show
- In job control and Job pipeline: the opening jobs should appear as "Opening" pill and will be subsequentely processed as normal jobs.
- In Job view: Transaction summary: the first transaction should show as "Opening Job" instead of "Received"
